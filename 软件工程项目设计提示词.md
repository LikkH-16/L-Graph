一、 项目名称
L-Graph: Learning Graph
——基于预置学科体系的智能导航式知识图谱笔记平台

二、 需求分析
1. 核心权限与角色 (新增)

多角色机制：系统包含普通用户与管理员角色。管理员配备专属后台管理模块，负责底层学科标准知识树字典的录入与维护（CRUD操作）。

2. 核心需求 (MVP版本)

学科知识字典预载：系统底层预录入考研（如408计算机基础、高等数学）等学科的标准化层级知识树。用户无需从零构建目录，直接在既定知识体系下挂载笔记。

层级化笔记管理 (修改)：支持 Markdown 格式编写，支持代码高亮与 LaTeX 数学公式。一个“笔记”代表一个学科的整体，笔记内细分为多个“页（Page）”来区分具体的知识点。

动态图谱与关系关联 (修改)：前端编辑器保存后，用户进入专属功能区，手动选择当前“页”所属的知识点，同一页可同时挂载到多个同层级知识节点下（多对多关系）。系统根据用户的选择与笔记间的双向引用提取边关系，自动生成可视化关系网。节点颜色根据“掌握程度”或“复习次数”实时变化。

智能关联推荐：翻阅笔记时，系统基于知识树深度自动推荐同层级的“兄弟知识点”或关联的“重点错题”。

3. 特色与拓展需求

沉浸式复习演示 (Slide Mode)：集成 Reveal.js，一键将 Markdown 笔记渲染成具有交互动画的网页版 PPT，支持自考式复盘。

学习进度可视化：图谱节点通过大小和颜色深浅展示复习进度（如一轮、二轮、三轮复习状态）。

AI 智能增强 (二期预留)：

自动标签挂载：AI 自动识别笔记内容并建议归属的知识节点。

本地大模型辅助：利用高性能硬件算力，通过本地部署的轻量级模型提供离线代码解析与错题讲解。

三、 设计思路
本系统采用“结构先行，数据填充”的设计哲学。不同于传统笔记系统的“自下而上”盲目记录，本系统强调“自上而下”的体系导航。

数据维度分离：将“学科标准知识树（静态）”与“个人笔记数据（动态）”解耦。

图关系驱动：不仅存储父子层级关系，更通过笔记间的引用构建跨章节的网状联系。

闭环反馈机制：通过“记录 -> 关联 -> 图谱查看 -> 演示复习 -> 进度反馈”形成完整的学习闭环。

四、 技术栈选型
1. 前端 (Client Side)

框架：Vue 3 + TypeScript (提高代码健壮性)

UI 库：Element Plus (响应式后台管理界面)

可视化引擎：AntV G6 (专门用于图谱渲染，支持大规模节点交互)

编辑器：Vditor / ByteMD (支持数学公式与代码块)

演示库：Reveal.js

2. 后端 (Server Side)

核心框架：Spring Boot (Java)

权限管理：Spring Security + JWT

持久层：MyBatis-Plus (快速开发 CRUD)

3. 数据库与环境 (Infrastructure) (修改)

双库混合存储架构：

MySQL 8.0：存储核心业务的结构化数据（如用户信息、权限配置、笔记纯文本内容等）。

Neo4j (图数据库)：专用于存储学科知识节点、笔记“页（Page）”实体，以及它们之间的多对多挂载关系和网状链接边关系，支撑高效的图关系遍历。

缓存：Redis (可选，用于加速知识树读取)

开发工具：IntelliJ IDEA (后端)、VS Code (前端)

运行环境：项目支持在高性能 PC 架构下流畅运行（充分利用大内存与高性能 CPU 进行图谱实时计算）。
