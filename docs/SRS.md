## 1. 引言

### 1.1 编写目的

本文档旨在完整描述 L-Graph 知识图谱笔记平台的功能需求和非功能需求，为后续的系统设计、编码实现和测试验收提供依据。本文档的预期读者包括项目开发人员、测试人员、项目管理人员以及最终用户。

### 1.2 读者对象

- **开发团队**：理解需求以进行系统设计和编码
- **测试团队**：根据需求编写测试用例
- **项目管理**：跟踪需求完成情况
- **终端用户**：了解系统功能范围

### 1.3 软件项目概述

**项目名称**：L-Graph（Learning Graph）
**简称**：LGraph
**开发单位**：LGraph 开发团队
**用户群体**：考研学生、自学者、知识管理需求者

**主要功能与用途**：
L-Graph 是一个面向计算机考研（CS 408）的知识图谱笔记平台。系统以学科为组织单位，以知识点树为骨架，支持用户编写 Markdown 笔记、将笔记页关联到知识点、记录复习进度，并通过知识图谱可视化掌握程度。系统区分普通用户和管理员角色，管理员可管理用户和全局数据。

**技术栈**：
- 前端：Vue 3 + TypeScript + Vite + Element Plus + AntV G6 + Reveal.js
- 后端：Spring Boot 3.2 + MyBatis-Plus + Spring Security + JWT
- 数据库：MySQL 8.0
- 基础设施：Docker Compose (MySQL + Neo4j + Redis)

## 2. 软件的一般性描述

### 2.1 软件产品与其环境之间的关系

L-Graph 采用 B/S 架构，前后端分离部署：

```
┌──────────────────────────────────────────────┐
│                    浏览器                      │
│         Vue 3 前端 (localhost:3000)            │
└──────────────────┬───────────────────────────┘
                   │ HTTP/REST (JSON)
                   │ JWT Bearer Token 认证
┌──────────────────▼───────────────────────────┐
│        Spring Boot 后端 (localhost:8080)       │
│              Context: /api/v1                  │
└──┬──────────────┬──────────────┬─────────────┘
   │              │              │
   ▼              ▼              ▼
┌──────┐    ┌──────────┐   ┌──────────┐
│MySQL │    │  Neo4j   │   │  Redis   │
│8.0   │    │  (预留)   │   │          │
└──────┘    └──────────┘   └──────────┘
```

**外部系统交互**：
- **Docker**：管理 MySQL、Neo4j、Redis 容器
- **Vite Dev Server**：开发时提供 HMR 热更新，代理 `/api` 到后端
- **Maven**：后端依赖管理和编译构建

### 2.2 限制与约束

| 约束类型 | 描述 |
|----------|------|
| 开发语言 | 后端 Java 21，前端 TypeScript |
| 数据库 | MySQL 8.0，字符集 utf8mb4 |
| 认证方式 | JWT 无状态认证，Token 有效期 24 小时 |
| 部署方式 | Docker Compose 容器化部署 |
| 浏览器兼容 | 现代浏览器（Chrome / Edge / Firefox 最新版） |
| 网络环境 | 前后端同机部署，无外部 API 依赖 |

### 2.3 假设与前提条件

1. 用户拥有基本的计算机操作能力
2. Docker Desktop 已正确安装并运行
3. 系统运行在本地开发环境，暂不考虑生产级安全加固
4. 知识体系数据（学科、知识点）由管理员预置，普通用户注册后直接使用
5. Neo4j 图数据库已配置但当前版本暂未使用（图谱基于 MySQL 实现）
6. Redis 已配置但当前版本暂未深度使用

#### 核心用例描述

**UC-01 用户注册**
- 参与者：未注册用户
- 前置条件：用户访问注册页面
- 主流程：填写用户名/邮箱/密码 → 提交 → 系统校验唯一性 → BCrypt 加密密码 → 创建 USER 角色账号 → 自动生成 JWT → 跳转学科列表
- 后置条件：用户已登录，localStorage 存储 JWT

**UC-02 用户登录**
- 参与者：已注册用户
- 前置条件：用户访问登录页面
- 主流程：输入用户名/密码 → 提交 → 查询用户 → BCrypt 比对密码 → 生成 JWT(含 userId, username, role) → 返回 token + 用户信息 → 跳转学科列表
- 后置条件：用户已登录
- 扩展：登录失败显示"用户名或密码错误"

**UC-03 编辑笔记页**
- 参与者：已登录用户
- 前置条件：已选择学科，已创建笔记
- 主流程：选择知识节点 → 创建/选择笔记页 → Markdown 编辑器编写内容 → 点击保存 → 内容持久化到数据库
- 后置条件：笔记页内容已保存

**UC-04 查看知识图谱**
- 参与者：已登录用户
- 前置条件：已进入学科详情页
- 主流程：点击"图谱"标签 → 系统获取个性化图谱数据(含用户 mastery) → G6 渲染 dagre 布局 → 节点颜色反映掌握程度
- 后置条件：图谱可视化展示

**UC-05 复习操作**
- 参与者：已登录用户
- 前置条件：学科下已有笔记页
- 主流程：进入"进度"页面 → 点击页面行的"复习"按钮 → 复习轮次+1，掌握等级+1(最高3级) → 复习记录写入数据库 → 页面 mastery 更新
- 后置条件：复习记录已保存，图谱中对应节点颜色更新

## 4. 其它软件需求描述

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 首屏 < 2 秒（Vite 构建产物 + CDN 加速的 Vditor/G6 等库） |
| API 响应时间 | 单次 CRUD < 200ms（本地 MySQL） |
| 图谱渲染 | 100 节点以下 < 1 秒（dagre 布局） |
| 并发用户 | 当前为单用户本地开发环境，未做并发测试 |
| 笔记编辑 | 实时输入延迟 < 50ms（Vditor IR 模式） |

### 4.2 设计约束

| 约束 | 说明 |
|------|------|
| 开发工具 | IntelliJ IDEA / VS Code |
| 运行环境 | JDK 21, Node.js ≥ 18, Docker Desktop |
| 安全性 | JWT + BCrypt，密码加密存储，HTTPS 未配置（本地开发） |
| 可靠性 | 数据持久化至 MySQL，Docker volume 持久化 |
| 可维护性 | 前后端分离，Pinia 状态管理，MyBatis-Plus ORM |
| 代码规范 | TypeScript 严格模式，Java Lombok 简化代码 |

### 4.3 界面要求

- **布局**：三栏式（左侧知识树 280px + 中间编辑区 + 右侧推荐面板可收拢）
- **主题**：支持暗色/亮色切换，默认暗色 `#020617` 深蓝背景
- **导航**：顶部面包屑（首页 → 学科 → 知识点）+ 视图标签（编辑/图谱/演示/进度）
- **响应式**：当前仅适配桌面端（≥1024px），移动端待后续支持
- **交互**：Element Plus 组件库统一交互风格

### 4.4 进度要求

项目当前处于 V1.0 开发阶段，已完成核心功能：
- ✅ 用户注册/登录/JWT 鉴权
- ✅ 学科浏览 + 三级知识树
- ✅ Markdown 笔记编辑 + 保存
- ✅ 知识图谱可视化

## 5. 软件原型

当前可运行的原型通过以下方式启动：

```bash
# 1. 启动基础设施
cd F:/Learning-Graph
docker compose up -d

# 2. 启动后端
cd backend
./mvnw spring-boot:run

# 3. 启动前端
cd frontend
npm run dev
```

访问 `http://localhost:3000`，使用预置账号 `admin / admin123`（管理员）或 `student / admin123`（普通用户）登录。
