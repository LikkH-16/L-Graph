# L-Graph 项目启动指南

知识图谱笔记平台 —— 全栈项目启动流程。

---

## 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Docker + Docker Compose | 最新 | 运行 MySQL / Neo4j / Redis |
| JDK | 21 | 后端编译运行 |
| Maven | 3.8+ | 后端构建（或使用 `./mvnw`） |
| Node.js | 18+ | 前端开发 |
| npm / pnpm | 最新 | 前端包管理 |

---

## 1. 启动基础设施（Docker）

在项目根目录下，启动 MySQL、Neo4j、Redis：

```bash
cd F:\Learning-Graph
docker-compose up -d mysql
```

> 仅启动 MySQL 即可运行（Neo4j 和 Redis 已配置但代码中尚未使用）。
> 如需全部启动：`docker-compose up -d`

**首次启动会自动执行**：
- `backend/src/main/resources/db/schema.sql` → 建表
- `backend/src/main/resources/db/seed.sql` → 插入 4 门学科 + 50+ 知识节点 + 2 个测试用户

**验证 MySQL 已启动**：
```bash
docker ps | grep mysql
```

---

## 2. 启动后端（Spring Boot）

```bash
cd F:\Learning-Graph\backend

# 方式一：Maven Wrapper（推荐，无需安装 Maven）
./mvnw spring-boot:run

# 方式二：IDE 直接运行 LGraphApplication.java
```

**配置概要**（`application.yml`）：
- 端口：`8080`
- 上下文路径：`/api/v1`
- MySQL：`localhost:3306`（root / root）
- JWT 有效期：24 小时

**验证后端已启动**：
```bash
curl http://localhost:8080/api/v1/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"admin123"}'
```

---

## 3. 启动前端（Vue 3 + Vite）

```bash
cd F:\Learning-Graph\frontend

# 安装依赖（首次）
npm install

# 启动开发服务器
npm run dev
```

**配置概要**（`vite.config.ts`）：
- 端口：`3000`
- API 代理：`/api` → `http://localhost:8080`（开发环境自动代理，无需处理跨域）
- Mock 模式：通过 `.env.development` 中的 `VITE_USE_MOCK` 控制

**验证前端已启动**：
浏览器打开 `http://localhost:3000`

## 前端 Mock 模式

在 `frontend/.env.development` 中设置：

```
VITE_USE_MOCK=true    # 使用前端 Mock 数据，无需后端
VITE_USE_MOCK=false   # 连接真实后端 API
```

Mock 模式下，前端使用内置的模拟数据运行，登录时密码任意填写即可（密码字段用于校验：统一密码为 `password`）。

## 数据库结构

所有数据存储在 MySQL 的 `lgraph` 数据库中，表结构：

```
lgraph_user              用户表
lgraph_subject           学科表（4门CS专业课）
lgraph_knowledge_node    知识节点树（CHAPTER > SECTION > TOPIC 三级）
lgraph_note              笔记（用户在某学科下的笔记）
lgraph_page              页面（笔记中的 Markdown 页面）
lgraph_page_knowledge_node  页面↔知识节点关联（M:N）
lgraph_review_record     复习记录
lgraph_mistake_note      错题本
```

物理文件位置：`F:\Learning-Graph\.data\mysql\lgraph\`（Docker 卷挂载）

## 常见问题

**Q: 启动后端报数据库连接失败？**  
确保 Docker MySQL 已启动：`docker ps | grep mysql`  
如未启动：`docker-compose up -d mysql`

**Q: 前端访问 API 404？**  
开发模式下 Vite 自动代理 `/api` 到后端 `8080` 端口，确保后端已启动。  
若使用 Mock 模式（`VITE_USE_MOCK=true`），前端不访问后端。

**Q: 登录失败？**  
检查 MySQL 中用户表是否已初始化种子数据。

**Q: 端口被占用？**  
`8080`（后端）：修改 `application.yml` 中的 `server.port`  
`3000`（前端）：修改 `vite.config.ts` 中的 `server.port`
